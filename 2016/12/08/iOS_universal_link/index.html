##iOS UniversalLink

####Step1:创建一个json 格式的apple-app-site-associatio 文件如下：

{
  "applinks": {
      "apps": [],
      "details": [
          {
              "appID": "9JA89QQLNQ.com.apple.wwdc",
              "paths": [ "/wwdc/news/", "/videos/wwdc/2015/*" ]
          },
          {
              "appID": "TeamID.BundleID2",
              "paths": [ "*" ]
          }
      ]
  }
}

“appID”组成部分：TeamID + BundleId TeamID可以从苹果开发账号页面也“Your Account”下查看，BundleId就直接在工程里看了

根据 paths 键设定允许的路径列表

"paths": [ "/wwdc/news/", "NOT /videos/wwdc/2010/*", "/videos/wwdc/201?/*"]

如果不需要处理某个路径下的UniversalLink，可在前面加”NOT “申明
注意：paths 路径是大小写敏感的

####Step2:上传 apple-app-site-association 文件
1、上传到web server根目录下或者.well-known子目录下

例:

https://<domain>/apple-app-site-association

or 

https://<domain>/.well-known/apple-app-site-association.

2、web server 需要支持https,客户端需要通告https访问，并且不支持任何重定向

####Step3:Xcode配置
1、添加域名到 Capabilities

在 Xcode 的 capabilities 里 添加你的 APP 域名, 前缀为 applinks: 。在APP第一次启动的时候，它会从 https://domain.com/apple-app-site-association 下载apple-app-site-association这个文件。

2、在 AppDelegate 里实现方法处理Action： 

- (BOOL)application:(UIApplication *)application continueUserActivity:(NSUserActivity *)userActivity restorationHandler:(void (^)(NSArray *restorableObjects))restorationHandler

当iOS通过链接打开APP时，会有一个类型为NSUserActivityTypeBrowsingWeb的NSUserActivity对象，这个对象的webpageURL属性就是用户访问的URL
